# Политика сложности пароля администратора

## Назначение

В проекте один вход в админ-панель по email и паролю ([`/admin/login`](../routes/web.php)). Отдельной формы смены пароля в интерфейсе нет; пароль задаётся при создании пользователя в таблице `users` (миграции, сидер, консоль).

## Что задано в коде

В [AppServiceProvider.php](../app/Providers/AppServiceProvider.php) настроено `Password::defaults()`:

- минимум **10** символов;
- обязательны **буквы в разном регистре** (mixed case);
- обязательны **цифры**.

Эти правила автоматически применяются там, где валидация использует `Password::defaults()` (например, будущая смена пароля или регистрация). Текущий вход через `AdminLoginController` проверяет только «непустой пароль» — **сила пароля определяется тем, как он задан в БД**.

## Рекомендации для production

1. Не использовать демо-пароль из документации на боевом сервере.
2. Задать уникальный длинный пароль (не менее требований выше), хранить только в менеджере паролей.
3. После первого входа при необходимости сменить пароль через `php artisan tinker` или отдельную процедуру, не коммитя секреты в git.

## Демо-окружение

Сидер [DatabaseSeeder.php](../database/seeders/DatabaseSeeder.php) создаёт учётную запись для локальной разработки; пароль указан в комментарии к сидеру и в [README.md](../README.md) только для тестовых стендов.
